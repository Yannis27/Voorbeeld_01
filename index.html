10 zaken waarom je echt moet overstappen op SASS

1. VARIABELEN

Sass biedt variabelen als een manier om informatie op te slaan die u opnieuw wilt gebruiken in uw stylesheet. Nu kunt u die kleurwaarde of lange lettertypestapel opslaan als iets dat u gemakkelijk kunt onthouden. De manier waarop u een variabele declareert is met een dollarteken, '$', gevolgd door de naam. Deze naam kan zijn zoals u wilt. Vervolgens typt u een dubbele punt gevolgd door de waarde en een puntkomma om het af te sluiten.

$mainFont: "Helvetica Neue", Arial, sans-serif;
$mainColor: #CC6699;

.mySelector { font-family: $mainFont; color: $mainColor; }

2. @IMPORT

Nu zeg je tegen jezelf: "CSS heeft @import, het is niet zo cool" en je hebt gelijk, maar de CSS- en Sass-versies verschillen op een significante manier. In normale CSS worden @import vanuit andere stijlbladen opgehaald, maar dit gebeurt door een ander HTTP-verzoek te doen, iets wat we meestal willen vermijden. Om deze reden heb je misschien nog nooit @import gebruikt. Sass, aan de andere kant, is een preprocessor (nadruk op de pre) die dat bestand zal ophalen voordat het de CSS compileert.

@import 'partials/myPartial';

3. KLEURFUNCTIES

Ik weet dat niet iedereen een programmeur is en dat het concept van een functie een beetje over je hoofd hangt, maar maak je geen zorgen, veel voegen een hoop nuttige functies toe zonder al te ingewikkeld te zijn. Wat kleuren betreft, zijn er verschillende handige om ze te manipuleren, maar drie vallen op als geweldige, makkelijke voordelen voor mensen die net zijn begonnen.

//syntax lighten($color, $amount) darken($color, $amount) rgba($color, $alpha)

//in parenthesis you can put any color value followed by the amount you want to modify it by.
//lighten(#000, 10%)
//darken(rgb(0,0,0), 25%)
//rgba(blue, 0.6)
//rgba($mainColor, 0.6)
//use case
$color: #333;//set color variable
.myButton {
background-color: rgba($color, 0.8);
color: lighten($color, 65%);
border: 1px solid darken($color, 5%);
}

//this compiles to:
.myButton {
background: rgba(51, 51, 51, 0.8);
color: #d9d9d9;
border: 1px solid #262626;
}

4. MIXINS

Sommige dingen in CSS zijn een beetje vervelend om te schrijven. Mixins maken groepen CSS-verklaringen die we opnieuw kunnen gebruiken op onze site. CSS3-stijlen waarvoor voorvoegsels van leveranciers nodig zijn, zijn een perfect voorbeeld van het gebruik van een mixin. In plaats van dezelfde eigenschap steeds opnieuw te typen, schrijven we een mixin eenmaal en noemen we die mix wanneer we deze willen gebruiken. Om een ​​mixin te declareren, gebruiken we het @mixinsleutelwoord.

//declare mixin(now being passed an argument)
@mixin box-sizing($boxSize) {
-webkit-box-sizing: $boxSize;
-moz-box-sizing: $boxSize;
box-sizing: $boxSize;
}

//use mixin
.mySelector {
@include box-sizing(border-box);
}

//compiled to:
.mySelector {
-webkit-box-sizing: border-box;
-moz-box-sizing: border-box;
box-sizing: border-box;
}

5. @EXTEND

@extend is een van de handigste functies waarmee we een reeks CSS-eigenschappen van de ene selector naar de andere kunnen delen. Denk aan een paar knoppen, zoals een knop voor accepteren en weigeren. Omdat ze beide knoppen zijn, delen ze waarschijnlijk het grootste deel van dezelfde stijl, maar de knop voor weigeren heeft een rode achtergrond om deze te laten opvallen. Met Sass schrijven we de standaardstijlen voor alle knoppen en "breiden" deze stijlen vervolgens uit naar de knop weigeren, waar we de rode achtergrond zouden toevoegen.

.button {
background: rgba($color, .8);
color: lighten($color, 65%);
border: 1px solid darken($color, 5%);
padding: 1em;
display: block;
max-width: 200px;
}
.button-decline {
@extend .button;
background: red;
}
//compiles to
.button, .button-decline {
background: rgba(51, 51, 51, 0.8);
color: #d9d9d9;
border: 1px solid #262626;
padding: 1em;
display: block;
max-width: 200px;
}
.button-decline {
background: red;
}

6. SNEL EN FLEXIBEL

Door middel van variabelen en andere trucjes kan men veel sneller code schrijven in Sass en kan men deze code veel makkelijker veranderen. Het zorgt er ook voor dat het designen veel flexibeler wordt wat ook een zeer groot pluspunt is.

7. MINDER CODE

Als er veel code wordt geschreven met herhalende dingen zoals bijvoorbeeld afbeeldingen kan de code sterk verkort worden. Als er weinig code wordt geschreven met veel unieke dingen maakt Sass de code alleen maar langer, een voorbeeld hiervan zijn "mixins".

8. NESTING

Nesting zorgt ervoor dat de code steeds overzichtelijk en geordend blijft.

9. MAKKELIJKE SET UP EN GEBRUIK

Hoewel het zoveel voordelen geeft tegenover de gewone CSS is het makkelijk te gebruiken. Dit komt omdat Sass eigenlijk bijna exact hetzelfde is als CSS. De setup is heel makkelijk aangezien het Sass bestand gewoon een kopie is van het huidige CSS bestand.
